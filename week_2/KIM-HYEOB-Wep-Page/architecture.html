AWS Web Server Architecture

위 아키텍처는 온라인 쇼핑몰을 위한 AWS 웹서버 아키텍처 구조를 제가 직접 설계한 것입니다. 아키텍처를 설계할 때 크게 3가지인 가용성, 안정성, 속도를 핵심으로 고려하여 설계하였습니다. 이와 더불어서 고객 맞춤형 서비스 제공을 위한 구성도 추가하였습니다.

먼저 3-tier(WEB-WAS-DB)로 구성해서 정적·동적 컨텐츠 제공에 대한 가용성을 높였습니다. 또한 ELB, 오토스케일링, RDS 복제를 통해서 수요가 급증할 때를 대비해 서버의 가용성을 높였습니다. DB는 RDS와 더불어 네트워크 성능이 안정적인 Dynamo DB를 도입함으로써 안정성을 높였습니다. CloudFront를 사용하여 웹 컨텐츠를 사용자에게 빠르게 배포하도록 했습니다. API와 고객 맞춤형 서비스 제공은 Lambda를 이용해서 서버리스 환경을 구축했으며, 특히 고객 맞춤형 서비스는 SQS, SNS, Dynamo DB를 통해 데이터를 수집하고, Glue를 통해 마이그레이션한 후, Personalize를 통해 데이터를 학습하여 유저에게 서비스를 제공하도록 했습니다. 또한 CloudWatch로 서비스를 관리하고 모니터링하고, 다양한 이메일 발송을 위해 SES을 사용했습니다.
